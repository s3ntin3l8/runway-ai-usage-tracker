# Antigravity Collector

**File:** `app/services/collectors/antigravity.py`

File-based quota collector for the Antigravity IDE.

## Overview

- **Collection Strategy**: LSP Probing → Local JSON file (fallback)
- **Cards**: 1-3 cards per model/credit type
- **Authentication**: None (auto-probes LSP process or reads local file)

## Setup Methods Quick Overview

The Antigravity collector obtains quota information using two main strategies:

1.  **LSP Probing (Primary)**:
    *   **Method**: Automatically detects and probes the active Antigravity Language Server Protocol (LSP) process.
    *   **Details**: Requires the Antigravity IDE to be running. See [Primary: LSP Probing](#primary-lsp-probing) below.

2.  **Local JSON Quota File (Fallback)**:
    *   **Method**: Reads usage data from the `quota.json` file generated by the Antigravity IDE.
    *   **Configuration**: The path to this file is auto-discovered, but can be overridden using the `ANTIGRAVITY_QUOTA_PATH` environment variable.
    *   **Details**: See [Secondary: Local JSON Quota File](#secondary-local-json-quota-file) below and [Configuration section](#configuration).

## Data Source

### Primary: LSP Probing
**Mechanism:** Runway detects running Antigravity IDE processes, identifies their Language Server Protocol (LSP) ports and CSRF tokens, and directly queries the LSP endpoint for user status, quota information, and available credits.
**Auth:** Uses internal tokens from the running LSP process.
**Behavior:** If LSP probing is successful, this is the preferred data source.

### Secondary: Local JSON Quota File
**Location:** Platform-specific (configurable via `ANTIGRAVITY_QUOTA_PATH`):
- **Linux:** `~/.local/share/antigravity/state/quota.json`
- **macOS:** `~/Library/Application Support/antigravity/state/quota.json`
- **Windows:** `%APPDATA%\antigravity\state/quota.json`
**Format:**
```json
{
  "models": {
    "claude-sonnet-4": {
      "remaining_percent": 75.5,
      "resets_at": 1775570736
    }
  }
}
```
**Behavior:** This data source is used as a fallback if LSP probing fails or no active LSP process is detected. It provides a snapshot of the last known quota from the IDE.

## Troubleshooting

### No Antigravity cards in dashboard
**Cause:** LSP process not found/probed, or local file not found.
**Fix:**
1.  Ensure the Antigravity IDE is running and actively being used.
2.  Verify the LSP process is running (check system process monitor).
3.  If still no cards, check the local quota file (fallback):
    -   **Linux:** `ls ~/.local/share/antigravity/state/quota.json`
    -   **macOS:** `ls ~/Library/Application\ Support/antigravity/state/quota.json`
    -   **Windows:** `ls %APPDATA%\antigravity\state\quota.json`
4.  Verify permissions for the local quota file: `chmod 644 <path-to-quota.json>`

### LSP connection failed / Timeout
**Cause:** Firewall, incorrect port, or LSP not listening on expected port.
**Fix:**
1.  Check firewall settings allowing local connections to Antigravity IDE.
2.  Restart Antigravity IDE.

### Shows 0% for all models (from local file)
**Cause:** Quota file has zero values.
**Fix:** Use models in IDE to generate usage data.
