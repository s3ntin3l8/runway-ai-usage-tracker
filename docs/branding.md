# Branding

Runway has **one canonical brand mark**, and every other surface is derived from
it. Edit the master, then run one command to regenerate the rest — never hand-edit
a derived asset.

## Canonical source

**`assets/logo.svg`** is the single source of truth. The mark is a glowing glass
capsule read as a *runway*, marked with a dashed white centerline (the runway-line
motif), on a premium dark disc.

## Derived surfaces

| Surface | Asset(s) | Derived how |
|---|---|---|
| Browser favicon | `webapp/public/favicon.svg` | Copied verbatim from `assets/logo.svg` |
| PWA / home-screen icons | `webapp/public/{pwa-64x64,pwa-192x192,pwa-512x512,maskable-icon-512x512,apple-touch-icon-180x180}.png`, `webapp/public/favicon.ico` | Generated from `favicon.svg` by `@vite-pwa/assets-generator` (config: `webapp/pwa-assets.config.ts`) |
| In-app logo | `webapp/src/components/layout/RunwayMark.tsx` | Hand-maintained **inline** SVG that mirrors the master (gradient capsule + dashed centerline), minus the heavy glow filters, so it stays cheap at 24px. Update by hand if the master's silhouette/colours change. |
| README logo | `README.md` (`assets/logo.svg`, `width=128`) | References the master directly |
| Sidecar tray icons | `sidecar_app/assets/icon_{ok,warn,err,paused}.png` | Generated by `sidecar_app/assets/generate_icons.py` — the same capsule + centerline silhouette tinted by the per-status colour |

## Regenerating

```sh
make logo
```

This copies the master to `webapp/public/favicon.svg`, regenerates the PWA raster
icons (`npm run generate-pwa-assets`), and regenerates the sidecar tray icons
(`generate_icons.py`, which uses Pillow from the venv). All outputs are committed —
they are **not** produced by `vite build`, so a production build never needs `sharp`.

`RunwayMark.tsx` is intentionally not auto-generated; adjust it by hand when the
master's shape or palette changes.

## Notes

- The brand palette: dark base `#09090b`/`#18181b`, capsule gradient
  `#60a5fa → #8b5cf6 → #d946ef`, white centerline. `theme_color`/`background_color`
  in the web manifest (and the `<meta name="theme-color">` in `index.html`) use the
  dark base `#09090b`.
- The maskable icon fills the full canvas with `#09090b` (configured in
  `pwa-assets.config.ts`) so an OS squircle/rounded mask never reveals transparent
  corners.
- The server serves all of these from `webapp/dist/` with no extra routing — the
  SPA catch-all passes dotted paths through, and `.webmanifest` already maps to
  `application/manifest+json`.
